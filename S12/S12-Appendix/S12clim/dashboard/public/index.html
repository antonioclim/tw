Seminar 12 – Redux Dashboard

Un „meta-starterkit” pentru predare: 4 paşi + API, cu pornire/oprire din interfaţă şi explicaţii detaliate.

Setup (Windows 11):
  1. Dezarhivaţi proiectul în Z:\tw\SxTEST\FAZA12\nextlab (sau alt director).
  2. Rulaţi install_all.bat (instalează dependenţele pentru dashboard, paşi şi API).
  3. Rulaţi run_dashboard.bat şi deschideţi http://localhost:3000.
Resurse: Ghid Step 1 (HTML) · Ghid Step 2 (HTML) · Ghid Step 3 (HTML) · Ghid Step 4 (HTML) · S12-teorie (RO, DOCX) · Ghid Postman (Notes API)

Control panel

Ghid rapid de predare

Recomandare de parcurs (şi pentru a evita „salturi” conceptuale): Step 1 (routing) → Step 2 (useReducer) → Step 3 (Redux sync + delete local) → Step 4 (Redux async + delete via server).

În VS Code, deschideţi folderul rădăcină. Pentru fiecare pas, explicaţi: ce fişiere atingem, de ce, ce efect are în UI şi care este „pattern-ul” general.

Notă: acest kit foloseşte proiecte Vite + React (pornire rapidă, fără warnings de dependenţe vechi), iar pentru async foloseşte redux-promise-middleware (conform materialelor video). În prezentare (DOCX) este discutat şi Redux Toolkit + createAsyncThunk ca abordare modernă.

Dashboard-ul porneşte/omoară procesele npm start pentru fiecare pas şi pentru API. Dacă un pas nu porneşte, verificaţi că dependenţele sunt instalate şi că porturile sunt libere.
